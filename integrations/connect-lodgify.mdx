PX Accounting connects to Lodgify with an API key from your Lodgify account settings.

<Frame caption="Selecting Lodgify on the PMS Connection card">
  ![Lodgify in the PMS picker](/images/screenshots/pms-picker-lodgify.png)
</Frame>

## Before you start

You will need:

- A Lodgify plan that includes public API access. Some plans do not - if the API key field is empty, you will need to contact Lodgify support to have one provisioned.
- Owner or admin login.

## Step 1 - locate your API key

1. Sign in to Lodgify.
2. Open **Settings** -> **Public API**.
3. Copy the **API key** shown on the page.

For background, see Lodgify's docs: [Authorization](https://docs.lodgify.com/docs/authorization).

If the API key field is empty:

- Contact Lodgify support and ask them to provision an API key for your account. Self-serve issuance is not always available.
- Confirm your plan includes API access.

<Warning>
  The Lodgify API key carries full read and write scope. Treat it as a production credential.
</Warning>

## Step 2 - paste the key into PX

1. In PX, open **Settings**.
2. In the **PMS Connection** card, pick **Lodgify**.
3. Paste the **API key**.
4. Click **Save credentials**.

PX validates the key and the **Integration Status** panel shows three lights: **Admin**, **OAuth**, and **Webhook**.

## What PX reads from Lodgify

PX has read-only access. It never writes back to Lodgify.

### Properties

- Property ID
- Name
- Full address (street, city, state, postal code, country)
- Bedrooms, bathrooms, max guests
- Property type
- Currency
- Active / inactive status
- Thumbnail picture

### Owners

Lodgify exposes a contact / owner record per property when configured. When present PX reads:

- Owner ID
- First name, last name
- Email address
- Phone number

### Reservations

- Booking ID and property reference
- Status (Booked, Tentative, Declined, Canceled)
- Check-in and check-out dates
- Booking date
- Guest first name, last name, email, phone, country
- Guest count
- Channel / source (Airbnb, Vrbo, Booking.com, direct, etc.)
- Confirmation code, external channel reservation number
- Currency
- Total amount, total paid, balance due, total refunded

### Per-charge financial data

- Per-line amount, type (accommodation, fee, tax, discount), description, taxable flag
- Per-night accommodation breakdown
- Per-reservation aggregates: subtotal, total taxes, channel fee, payout
- Payment quotes and payment records: amount, date, method, status

### Tax configuration

PX reads tax settings directly from Lodgify: rates, names, applicability per property. Channel sync overrides apply on top inside PX.

### Webhook events

PX subscribes to Lodgify webhooks for:

- Booking created, updated, canceled
- Property created, updated
- Payment received, refunded

PX is read-only against Lodgify even though the API key technically carries write scope.

## Removing the connection

Open **Settings**, scroll to the **PMS Connection** card, and click **Delete**. The key is wiped from PX and webhooks are unregistered.

## Troubleshooting

**Admin status is red.**
The API key is wrong or has been rotated. Pull the latest value from Lodgify and update PX.

**API key field is empty in Lodgify.**
Contact Lodgify support to have one provisioned.

**Webhook status is red.**
Verify your Lodgify plan includes webhook delivery. Click **Refresh** in PX after confirming.
