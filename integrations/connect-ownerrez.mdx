OwnerRez uses a one-click OAuth flow. There is nothing to copy and paste - you click a button in PX, sign in to OwnerRez, and grant access.

## Before you start

You will need:

- An OwnerRez account with API access enabled (most plans include this; contact OwnerRez support if API is locked).
- An owner-level login - sub-users may not have permission to authorize third-party apps.

## Step 1 - authorize PX

1. In PX, open **Settings**.
2. In the **PMS Connection** card, pick **OwnerRez**.
3. Click **Connect with OwnerRez**.
4. Sign in to OwnerRez when prompted.
5. Review the permissions and click **Allow**.

OwnerRez redirects you back to PX. The **Integration Status** panel now shows green lights for **Admin**, **OAuth**, and **Webhook**.

## Step 2 - build the OwnerRez catalog

This step is unique to OwnerRez. **You must do it before audits return useful results.**

OwnerRez has no API to read your tax, surcharge, or commission setup. Instead, PX:

1. Scans your last 18 months of bookings.
2. Infers a catalog of taxes, fees, and commission rates from the line items it sees.
3. Asks you to confirm each entry.

To do this:

1. Open **PMS Configuration** in the sidebar.
2. The **OwnerRez Catalog** loads with inferred entries marked `inferred`.
3. Click **Detect from bookings** if you want PX to (re-)scan history.
4. Walk through each row and:
   - Click **Confirm** if the inferred rate matches what you actually charge.
   - **Edit** the rate, unit (percent vs fixed), or applicability scope if needed.
   - Click **Add row** for any tax or fee PX did not pick up.
   - **Archive** entries that are out of date.

Entries can be in one of four statuses:

- **inferred** - PX detected this from booking history. Not yet used by audits.
- **user-added** - you created it manually.
- **confirmed** - reviewed and approved. Used by audits.
- **archived** - hidden, not used.

Audits use **confirmed** and **user-added** rows only. If the catalog is empty, audits short-circuit with a `CATALOG_EMPTY` finding instead of running.

See [OwnerRez Catalog](/pms-configuration/ownerrez-catalog) for the full UI walkthrough.

<Tip>
  Where do you actually configure these rates? On OwnerRez itself - **Settings -> Taxes** (account-wide), **Settings -> Surcharges** (fees), and **Property Management -> Setup & Configuration** (owner commissions). PX's catalog is your audit baseline; OwnerRez is the system of record.
</Tip>

## What PX reads from OwnerRez

PX has read-only access. It never writes back to OwnerRez.

### Properties

- Property ID
- Name
- Full address (street1, street2, city, state, postal code, country)
- Bedrooms, bathrooms, max guests
- Currency
- Owner ID (mapped to the Owner record below)

### Owners

- Owner ID
- First name, last name (parsed from the full name when needed)
- Email address
- Properties owned (list of property IDs)

### Reservations (bookings)

- Booking ID and property reference
- Status (Active, Pending, Canceled, Quote - mapped to PX's confirmed / inquiry / canceled / pending)
- Check-in and check-out dates
- Guest ID, guest name (enriched from `/v2/guests/{id}` when available)
- Guest count
- Channel / listing site (Airbnb, Vrbo, Booking.com, direct, etc.)
- Platform reservation number (the channel's confirmation, when present)
- Confirmation code
- Currency
- Total amount, total paid, balance due, total refunded, total host fees
- Booked date, created and updated timestamps
- Owner ID
- Block flag (true for blocked dates rather than guest stays)
- Canceled date, cancellation policy snapshot

### Per-charge financial data

For each charge on a booking:

- Amount, type (accommodation / surcharge / tax / expense), description
- Per-charge owner-vs-commission split: owner amount, commission amount, commission percent
- Taxable flag
- Source flags: inherited from account vs. property-overridden, channel-managed flag, raw position

For each payment, refund, and deposit (read separately):

- **Payments** - payment ID, booking, amount, collected date, type, deposited flag, total fees.
- **Refunds** - refund ID, booking, amount, paid date, deposited flag, linked payment ID.
- **Deposits** (payouts from the processor) - deposit ID, deposit date, processor, status, external deposit number, totals, individual transactions, fees, and adjustments.

### Tax configuration

OwnerRez's API does not expose your tax catalog, so PX:

1. Scans the last 18 months of bookings.
2. Infers a catalog of taxes, fees, and commissions from the line items it sees.
3. Asks you to confirm each entry.

You manage this on the **PMS Configuration** page under **OwnerRez Catalog**. The OwnerRez UI itself remains the system of record - PX's catalog is your audit baseline.

### Webhook events

PX subscribes to OwnerRez webhooks:

- `entity_create` / `entity_update` on **booking** - re-ingests charges, payments, refunds, and deposits for that booking.
- `entity_create` / `entity_update` on **property** - re-fetches the property.
- `entity_create` / `entity_update` on **payment**, **refund**, or **deposit** - re-ingests financials for the linked booking.
- `entity_delete` on booking, property, guest, or inquiry - removes the record from PX.
- `application_authorization_revoked` - tears down the OAuth credentials cleanly.

## Removing the connection

Open **Settings**, scroll to the **PMS Connection** card, and click **Delete**. This revokes the OAuth token, unregisters webhooks, and leaves your existing PX data in place.

## Troubleshooting

**OAuth pop-up does not return to PX.**
Pop-up blockers are a common cause. Allow pop-ups for `pxapp.net` and try again.

**Webhook status is red.**
OwnerRez may have rejected the webhook URL. Verify your OwnerRez plan includes webhook access and click **Refresh**.

**Audits return no results.**
The OwnerRez catalog is probably empty. Open **PMS Configuration**, run **Detect from bookings**, and confirm at least the taxes and any commission rules you charge.

**A booking has no inferred catalog rules.**
This is expected for one-off charges (cleaning fee on a single trip, manual adjustment). PX only infers rules that repeat. Add custom rows in the catalog for anything systematic.
